# kTeam Agent v3 — Release Notes

## 一句话总结

kTeam Agent 新增讨论总结、异常告警、任务管理和预算管理四大能力——现在可以在 Slack 中总结会议讨论并归档到 Notion，自动提醒超期未处理事项，对话式创建和更新任务，以及管理团队预算和支出。

---

## 已有功能（v2）

- 对话式提交反馈（Bug / Feature Request / Improvement）
- Notion 数据查询（任务、报销、招聘、反馈）
- 对话式报销 + 审批 + 付款全流程
- 对话式录入候选人 + Interviewer 通知
- Daily / Weekly 自动推送

## 新增功能

### 1. 讨论固化 / 会议纪要

在频道中 @Agent 即可总结讨论内容，支持按时间、话题、发言人灵活过滤，生成结构化总结后一键保存到 Notion。

**应用场景**
- 技术讨论结束后，@Agent「总结今天关于 agent 架构的讨论」，自动生成包含背景、要点、决策、待办的结构化文档
- 会议结束后，@Agent「总结今天的讨论」，快速生成会议纪要
- 想看某人的发言要点，@Agent「总结今天 @hcs 的发言」，只提取该用户的消息进行总结

**解决的问题**
- 讨论结论不再散落在聊天记录中，第一时间结构化归档到 Notion
- 支持时间 + 话题 + 用户的组合过滤，精准定位需要总结的内容
- 总结后自动弹出「Save to Notion」按钮，用户确认后写入，避免误操作
- 文档自动关联作者和分类，方便后续检索

**使用方式**
- `@agent 总结今天的讨论` — 总结当天所有讨论
- `@agent 总结今天关于 agent 的讨论` — 按话题过滤总结
- `@agent 总结今天 @username 的发言` — 按发言人过滤总结
- `@agent 总结本周的讨论` — 按时间范围总结

### 2. 异常告警

自动检测超过 3 天未处理的报销、招聘和反馈事项，定时推送告警到 #dashboard 频道，确保重要事项不被遗忘。

**应用场景**
- 有报销提交了 5 天还没审批，告警推送提醒审批人处理
- 有候选人等待面试安排超过 3 天，告警推送提醒 HR 跟进
- 有客户反馈提交了一周还没处理，告警推送提醒负责人

**解决的问题**
- 超期事项不再被遗忘，系统自动检测并推送
- 告警消息包含 Notion 链接、提交人、超期天数，一目了然
- 分类展示（报销待审批、报销待付款、招聘未处理、反馈未处理），便于快速定位
- 无超期项时不推送，避免信息噪音

**告警规则**
- 报销待审批：提交超过 3 天未审批
- 报销待付款：审批通过超过 3 天未付款
- 招聘未处理：候选人状态为 Pending Review 超过 3 天
- 反馈未处理：反馈状态为 Pending 超过 3 天

### 3. 任务管理

在 Slack 中对话式创建任务、更新任务进度、生成任务进度表，所有操作实时同步 Notion Tasks 数据库。

**应用场景**
- CTO 在群里说「创建一个任务：B1，名称：PG schema 设计，分配给 @hcs，截止日期：4月1日，优先级高」，Agent 整理后创建到 Notion，被分配人收到 DM 通知
- 开发同事汇报进度「更新任务 B1，进度：wa-bridge.ts 落库，实机验证通过」，Agent 自动追加到任务日志
- 每日站会后，@Agent「生成今天的任务进度表」，自动生成 Markdown 表格并同步到 Notion

**解决的问题**
- 任务创建不再需要打开 Notion 手动填写，一句话搞定
- 任务编号（如 B1、C3）自动识别，更新时精准匹配
- 人员分配支持 @mention、姓名、邮箱三种方式，自动映射到 Notion 用户
- 进度更新自动追加到日志（带时间戳），保留完整历史
- 状态自动推断：说「完成」「done」「100%」自动标记 Done，否则标记 In Progress
- 进度表自动同步到 Notion Docs，生成后分享链接

**使用方式**
- `创建一个任务：B1，名称：PG schema 设计，分配给 @hcs，截止日期：4月1日，优先级高` — 创建任务
- `更新任务 B1，进度：wa-bridge.ts 落库，实机验证通过` — 更新进度
- `更新任务 C1，已完成` — 标记任务完成
- `@agent 生成今天的任务进度表` — 生成当日进度表
- `@agent 生成本周的任务进度表` — 生成本周进度表

**进度表示例**

| 日期 | # | 任务 | 状态 | 今日进展 |
|---|---|---|---|---|
| 03-31 | B1 | PG schema 设计 | 🔵 进行中 | wa-bridge.ts 落库，实机验证通过 |
| 03-31 | C1 | API 接口开发 | ✅ 完成 | 全部接口联调通过 |
| 03-30 | A2 | 需求文档 | ⬜ 未开始 | - |

**总进度：1/3 完成（33%）**

### 4. 预算管理

在 Slack 中对话式管理团队预算和支出，支持更新预算、添加支出、查询预算状态，所有数据实时同步 Notion。

**应用场景**
- 财务说「更新预算，人力资源，1000」，Agent 直接更新 Notion 中对应分类的月度预算
- 采购后说「添加支出，MacBook，200」，Agent 自动推断分类为 Equipment Purchases，确认后记录
- 月末查看「查看本月人力资源预算」，Agent 返回预算金额、已花费、占比
- 查看全局「本月总支出」，Agent 返回所有分类的汇总数据

**解决的问题**
- 预算更新不再需要打开 Notion 手动修改，一句话搞定
- 支出分类由 Agent 根据描述自动推断（MacBook→设备、打车→交通差旅、午饭→餐饮）
- 支持中英文分类名（人力资源→Human Resources），Agent 自动映射
- 月份自动识别，支出自动关联当月
- 报销审批通过后自动同步到支出数据库，预算占比实时更新
- 查询结果包含预算/已花费/占比，一目了然

**预设预算分类**
- Human Resources（人力资源）
- Rent（房租）
- Living Expenses（生活费）
- Visa Costs（签证成本）
- Materials（物料）
- Equipment Purchases（设备购买）
- Miscellaneous（杂费）
- Transportation & Travel（交通差旅）
- Client Entertainment（客请费用）

**使用方式**
- `更新预算，人力资源，1000` — 更新预算金额
- `添加支出，MacBook，200` — 添加支出（自动推断分类）
- `查看本月人力资源预算` — 查询单分类预算状态
- `本月总支出` — 查询全部分类汇总
- `查看本月设备支出` — 查询分类下的支出明细
