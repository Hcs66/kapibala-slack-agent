# Kapibala Slack Agent v2 — Release Notes

## 一句话总结

kTeam Agent 从「只能读 Slack 消息的助手」升级为「团队操作中枢」——现在可以直接在对话中提交反馈、报销、录入候选人，并查询 Notion 中的任务和审批状态。

---

## 新增功能

### 1. 对话式提交反馈

在 Slack 中用自然语言描述 bug 或需求，Agent 自动提取结构化信息，确认后一键写入 Notion。

**应用场景**
- 开发同事在群里说「登录页点确认没反应，客户 A 反馈的，挺急的」，Agent 自动识别为 P1 Bug，整理好字段让你确认后直接提交
- 产品经理随手提一句「加个暗黑模式吧，不急」，Agent 归类为 P3 Feature Request

**解决的问题**
- 不再需要打开 Notion 手动填表，也不需要记住各字段的选项
- Priority、Type、Source 由 Agent 根据语义自动推断，减少人工判断成本
- 提交后自动通知到反馈频道，团队实时可见

### 2. Notion 数据查询

直接在 Slack 中问 Agent「报销还有几笔没处理」「招聘管线什么情况」，即时获得格式化的结果摘要。

**应用场景**
- 管理者查看全局：「目前有多少待审批的报销？」「Software Engineer 岗位有几个候选人？」
- 不用切换到 Notion，在 Slack 里就能完成信息获取

**解决的问题**
- 团队成员不需要学习 Notion 的筛选和视图操作
- 查询结果自带 Notion 链接，需要详情时一键跳转
- 支持按类型、优先级、状态等维度过滤

### 3. 对话式报销 + 审批

在 Slack 中说「我要报销上周打车 150 AED」，Agent 追问缺失信息后提交到 Notion，同时推送审批消息到指定频道，审批人一键操作。

**应用场景**
- 出差回来在 Slack 里随手报销，不用打开 Notion 填表
- 审批人在 #expense-claims 频道直接点按钮审批，提交者收到 DM 通知结果
- 提交后可上传发票附件，自动同步到 Notion 页面

**解决的问题**
- 报销流程从「打开 Notion → 找到数据库 → 填写表单 → 通知审批人」简化为一句话
- 审批不再依赖 Notion 内操作，在 Slack 中闭环完成
- 审批状态实时同步回 Notion，避免信息不一致

### 4. 对话式录入候选人

在 Slack 中提到候选人信息，Agent 自动提取姓名、职位、来源等字段，确认后写入 Notion 招聘数据库。

**应用场景**
- HR 或面试官在群里说「有个候选人叫张三，应聘 Software Engineer，LinkedIn 上看到的」，Agent 整理后直接录入
- 录入后可上传简历附件，自动关联到 Notion 页面
- 提交后自动通知招聘频道，团队同步知晓

**解决的问题**
- 候选人信息不再散落在聊天记录中，第一时间结构化入库
- 简历附件与候选人记录自动关联，不用手动上传到 Notion
- 减少 HR 的重复录入工作

## 后续更新计划

| 版本 | 功能 | 说明 |
|---|---|---|
| v2.1 | 讨论固化 / 会议纪要 | 在 thread 里 @Agent「帮我总结这个讨论」，自动生成 Decision Record 写入 Notion |
| v2.2 | Daily / Weekly 推送 | 定时从 Notion 聚合数据，推送周报和日报到指定频道 |
| v2.3 | 异常告警 | 任务超期、报销长时间未审批、项目无更新时自动提醒相关负责人 |
| v3.0 | 产品知识 RAG | Agent 能回答「代码规范是什么」「部署流程」等团队知识库问题 |
| v3.1 | 任务分配 | CTO 可通过 Agent 直接分配任务，被分配人收到 DM 通知 |
| v3.2 | 面试日历 + 提醒 | 集成 Google Calendar，支持安排面试和自动提醒 |
